01 — DETECTION PIPELINE

Every script gets checked twice, before it ever runs.

The page is intercepted at two separate layers — one stops known threats from connecting at all, the other watches the live DOM for anything that slips through.

STEP 01
Network interception
Before a request to a known-bad domain even leaves the browser, Chrome's network layer kills it — no script downloaded, no connection made.
declarativeNetRequest → block()
STEP 02
Live DOM surveillance
A MutationObserver watches every node added to the page. New <script> or <iframe> tags are matched against the pattern engine the instant they appear.
MutationObserver → matchThreats()
STEP 03
Removal & reporting
Matches are stripped from the page before execution. The finding is sent to the background worker, which updates the badge count for that tab.
node.remove() → reportToBackground()
STEP 04
Risk scoring
The popup aggregates every finding into a weighted risk score — critical hits count for more than a stray tracker — and renders it as a single number.
critical×18 + high×10 + medium×5

04 — UNDER THE HOOD

No ML model. No server. Nothing to leak.

Everything below runs locally inside the browser tab. There's no API call carrying your browsing history anywhere — which is the entire point of a privacy tool.

Engine
Vanilla JavaScript (ES6+) — regex pattern matching, zero dependencies
Platform
Chrome Extensions API, Manifest V3
Network layer
declarativeNetRequest — 20 domain-level block rules
DOM layer
MutationObserver — real-time node interception
Interface
HTML/CSS popup with animated risk gauge
Storage
chrome.storage.local — theme preference only, nothing else persisted
